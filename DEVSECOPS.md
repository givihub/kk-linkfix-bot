# DevSecOps — kk-linkfix-bot

Журнал мер безопасности проекта. Обновляется при каждом значимом изменении.

## Управление секретами

- Токен бота хранится **только** в `.env` на сервере (права `600`, владелец `givx`). В репозитории — только `.env.example` с плейсхолдером.
- `.gitignore` исключает `.env` и любые `*.env`, кроме `.env.example`.
- Токен не пишется в логи; логируются только id чатов/пользователей и количество ссылок.
- При компрометации токена: `/revoke` в @BotFather → обновить `.env` → `docker compose up -d`.

## Контейнер (Dockerfile / docker-compose)

- Базовый образ — минимальный `python:3.12-slim`, зависимости пинованы (`aiogram==3.30.0`).
- Процесс работает от **непривилегированного пользователя** `botuser` (не root).
- `read_only: true` — файловая система контейнера только на чтение (запись только в tmpfs `/tmp`).
- `no-new-privileges:true` — запрет эскалации привилегий.
- `cap_drop: ALL` — сброшены все Linux capabilities.
- Логи ограничены (10 МБ × 3 файла) — защита от переполнения диска.
- `restart: unless-stopped` — автоподъём после сбоя/перезагрузки сервера.

## Сеть

- Бот работает по **long polling** — не открывает ни одного входящего порта, вебхуков и TLS-терминации нет, поверхность атаки со стороны интернета нулевая.
- `network_mode: host` используется только ради доступа к локальному Xray-прокси хоста (`127.0.0.1:7890`) для api.telegram.org (прямой доступ из РФ недоступен). Наружу бот ходит только к Telegram API через этот прокси.
- Домены-фиксеры видео-превью (`kkinstagram.com`, `kktiktok.com`, `fixupx.com`) опрашивают серверы Telegram при построении превью.
- Для подписи (автор + текст поста) бот делает **исходящие GET-запросы** к caption-доменам (`tnktok.com`, `fixupx.com`) через тот же прокси: таймаут 4 с, читается максимум 256 КБ, только Content-Type text/html, любой сбой = отправка без текста (fail-soft). Никакие данные пользователей туда не передаются — только путь исходной ссылки.

## Минимизация данных

- Бот не хранит никаких данных: нет БД, нет файлов состояния, сообщения не логируются (только метаданные).
- Обрабатываются только сообщения, содержащие подходящие ссылки; остальные игнорируются.
- Скачивания контента нет — значит нет и хранения чужого контента на сервере.

## Права бота в Telegram

- Режим «заменить»: боту выдаётся ровно одно админ-право — «Удаление сообщений» (принцип наименьших привилегий). Прочие права не запрашиваются. Без права на удаление бот мягко деградирует: замена приходит, оригинал остаётся.

## Риски и их принятие

| Риск | Оценка | Митигция |
|---|---|---|
| Смерть домена-фиксера | Средняя вероятность, низкое влияние | Домены в `.env`, замена за минуту без пересборки |
| Компрометация токена | Низкая вероятность | Хранение только в `.env` (600), revoke-процедура |
| Недоступность Xray-прокси | Низкая | Бот переподключается сам; мониторинг прокси на стороне хоста |
| Спам-флуд ссылками в группе | Низкая | Обрабатываются только валидные ссылки; троттлинг Telegram API на стороне aiogram |

## История изменений

- **2026-07-23** — три защитных исправления: (1) редирект фиксера обратно на домен соцсети трактуется как отказ, а не как ссылка на видео — цепочка продолжается; (2) санити-чек скачанного файла (сигнатура mp4 `ftyp`, минимальный размер) — HTML-страницы и ошибки не загружаются в чат под видом видео; (3) исходное сообщение пользователя удаляется только при реально доставленном видео-файле — при откате на превью оригинал сохраняется, ссылка не теряется. Также добавлено подробное логирование резолвера (URL + ответ каждого фиксера) для разбора инцидентов.
- **2026-07-23** — фиксеры превращены в цепочки с автоматическим фолбэком (`*_FIX_DOMAINS`, порядок = приоритет): Instagram kkinstagram→vxinstagram, TikTok kktiktok→tnktok→a.tnktok, X d.fixupx→d.fxtwitter→d.vxtwitter. Резолвер понимает redirect и og:video. Смерть одного сервиса больше не роняет функцию.
- **2026-07-23** — доставка видео: бот скачивает файл через прокси (только в память, лимит 45 МБ, таймаут 120 с, ничего не сохраняется на диск) и загружает в Telegram как полноценное видео-сообщение. Причина: серверы Telegram не могут скачивать с CDN Instagram напрямую (sendVideo по URL и веб-превью флакают). При сбое/превышении лимита — откат на веб-превью.
- **2026-07-23** — переход в режим «заменить»: бот удаляет исходное сообщение (нужно право «Удаление сообщений») и публикует видео с подписью, строкой «от кого» и инлайн-кнопкой на оригинал.
- **2026-07-22** — подпись обогащена автором и текстом поста из OG-метатегов caption-доменов (TikTok: tnktok.com, X: fixupx.com; Instagram недоступен — kkinstagram отдаёт mp4 без метаданных). Появились контролируемые исходящие запросы: таймаут, лимит объёма, fail-soft; отражено в разделе «Сеть».
- **2026-07-22** — переход на режим «реплай» без удаления сообщений: с бота сняты админ-права в группе, поверхность привилегий сокращена до обычного участника.
- **2026-07-22** — добавлена платформа X/Twitter (fixupx.com); Reddit добавлялся и был убран по решению владельца. Принцип прежний: бот сторонние домены не посещает, превью строит Telegram.
- **2026-07-22** — первичная настройка: контейнер (non-root, read-only, cap_drop), секреты в `.env`, пинование зависимостей, деплой на FreeTier через прокси.
