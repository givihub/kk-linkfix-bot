# DevSecOps — kk-linkfix-bot

Журнал мер безопасности проекта. Обновляется при каждом значимом изменении.

## Управление секретами

- Токен бота хранится **только** в `.env` на сервере (права `600`, владелец `givx`). В репозитории — только `.env.example` с плейсхолдером.
- `.gitignore` исключает `.env` и любые `*.env`, кроме `.env.example`.
- Токен не пишется в логи; логируются только id чатов/пользователей и количество ссылок.
- При компрометации токена: `/revoke` в @BotFather → обновить `.env` → `docker compose up -d`.

## Контейнер (Dockerfile / docker-compose)

- Базовый образ — минимальный `python:3.12-slim`, зависимости пинованы (`aiogram==3.30.0`).
- Процесс работает от **непривилегированного пользователя** `botuser` (не root).
- `read_only: true` — файловая система контейнера только на чтение (запись только в tmpfs `/tmp`).
- `no-new-privileges:true` — запрет эскалации привилегий.
- `cap_drop: ALL` — сброшены все Linux capabilities.
- Логи ограничены (10 МБ × 3 файла) — защита от переполнения диска.
- `restart: unless-stopped` — автоподъём после сбоя/перезагрузки сервера.

## Сеть

- Бот работает по **long polling** — не открывает ни одного входящего порта, вебхуков и TLS-терминации нет, поверхность атаки со стороны интернета нулевая.
- `network_mode: host` используется только ради доступа к локальному Xray-прокси хоста (`127.0.0.1:7890`) для api.telegram.org (прямой доступ из РФ недоступен). Наружу бот ходит только к Telegram API через этот прокси.
- Сторонние домены-фиксеры (`kkinstagram.com`, `kktiktok.com`) бот **не посещает** — их опрашивают серверы Telegram при построении превью. Утечки IP сервера или данных пользователей в сторонние сервисы нет.

## Минимизация данных

- Бот не хранит никаких данных: нет БД, нет файлов состояния, сообщения не логируются (только метаданные).
- Обрабатываются только сообщения, содержащие подходящие ссылки; остальные игнорируются.
- Скачивания контента нет — значит нет и хранения чужого контента на сервере.

## Права бота в Telegram

- Боту выдаётся ровно одно админ-право в группе: «Удаление сообщений». Остальные права не запрашиваются (принцип наименьших привилегий).
- Если права на удаление нет — бот деградирует мягко: оставляет оригинал и просто добавляет сообщение с видео.

## Риски и их принятие

| Риск | Оценка | Митигция |
|---|---|---|
| Смерть домена-фиксера | Средняя вероятность, низкое влияние | Домены в `.env`, замена за минуту без пересборки |
| Компрометация токена | Низкая вероятность | Хранение только в `.env` (600), revoke-процедура |
| Недоступность Xray-прокси | Низкая | Бот переподключается сам; мониторинг прокси на стороне хоста |
| Спам-флуд ссылками в группе | Низкая | Обрабатываются только валидные ссылки; троттлинг Telegram API на стороне aiogram |
